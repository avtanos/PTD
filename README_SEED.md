# Скрипт заполнения базы данных тестовыми данными

## Описание

Скрипт `seed_projects.py` предназначен для заполнения базы данных тестовыми данными для модуля "Проекты".

## Использование

```bash
cd backend
python seed_projects.py
```

## Что создается

### Подразделения (5 штук):
1. **ПТО** - Производственно-технический отдел
2. **ОГЭ** - Отдел главного энергетика
3. **ОГМ** - Отдел главного механика
4. **Геодезия** - Геодезический отдел
5. **Склад** - Складское хозяйство

### Проекты (12 штук):
1. **PROJ-001** - Жилой комплекс "Солнечный" (Бетонные, активный)
2. **PROJ-002** - Офисный центр "Бизнес Парк" (Бетонные, активный)
3. **PROJ-003** - Реконструкция моста через р. Волга (Металл, активный)
4. **PROJ-004** - Торговый центр "МегаМолл" (Кладка, активный)
5. **PROJ-005** - Ремонт фасада административного здания (Отделка, активный)
6. **PROJ-006** - Благоустройство парка Победы (Благоустройство, приостановлен)
7. **PROJ-007** - Прокладка инженерных сетей для микрорайона (Сети, активный)
8. **PROJ-008** - Строительство школы на 500 мест (Бетонные, завершен)
9. **PROJ-009** - Реконструкция стадиона (Бетонные, активный)
10. **PROJ-010** - Устройство кровли производственного цеха (Кровля, черновик)
11. **PROJ-011** - Земляные работы под фундамент (Земляные, активный)
12. **PROJ-012** - Монтаж металлоконструкций склада (Металл, активный)

## Особенности

- Скрипт проверяет существование данных перед созданием (не создает дубликаты)
- Автоматически создает недостающие колонки в таблице `projects` (если таблица была создана ранее)
- Использует raw SQL для избежания проблем с relationships в SQLAlchemy
- Безопасен для повторного запуска (idempотентен)

## Требования

- Python 3.9+
- Установленные зависимости из `requirements.txt`
- База данных SQLite (создается автоматически при первом запуске приложения)
